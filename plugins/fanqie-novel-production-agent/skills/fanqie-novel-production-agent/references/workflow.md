# 番茄小说自动生产工作流

## 目录

0. 使用者配置
1. 输入与安全边界
2. 选题与人工确认
3. 策划冻结
4. 逐章生产与并行 Word
5. GPTs 全书复核
6. 合并文件
7. 番茄草稿
8. 封面与元数据
9. 完成与发布保护

## 0. 使用者配置

1. 按 [configuration.md](configuration.md) 的优先级定位每位使用者自己的配置文件。
2. 运行 `settings.py validate`；不存在或无效时，进入一次性的设置关卡。
3. 从配置读取小说作者名、输出根目录、GPTs 地址、非敏感账号提示、番茄工作台地址和 Word 模板路径。
4. 使用当前 Chrome 登录会话。账号提示与页面显示不一致时暂停确认，不保存或索取密码、Cookie、令牌。
5. 将已验证配置文件路径写入新书断点；共享插件时不得包含真实配置。

## 1. 输入与安全边界

1. 使用 Chrome 打开配置中的 GPTs 地址，并保持同一个 GPTs 对话。
2. 将完整的小说底层逻辑工程包 DOCX 上传给 GPTs。
3. 初始化书籍断点文件；在尚无正式书名时使用 `待定书名-日期时间` 建立临时目录，冻结书名后用 `checkpoint.py relocate` 安全迁移到正式书名目录并更新所有内部路径。
4. 记录工程包路径、GPTs 对话 URL、配置中的输出根目录和 Word 模板路径。
5. 所有书籍文件必须保存到配置的 `output_root`。
6. `word.template_path` 非空时使用该模板；留空时使用稳定默认模板。
7. 禁止在没有用户新授权的情况下提交审核、发布、签约或公开内容。

## 2. 选题与人工确认

1. 要求 GPTs 根据工程包生成一整组选题方案，暂时不写大纲或正文。
2. 将 GPTs 返回的全部选题原样展示给用户，不删减、不改写、不代选。
3. 将状态推进到 `awaiting_topic_confirmation`，写明下一步是等待用户选择。
4. 必须暂停。
5. 用户选择后，记录原始选择文本，并发回同一个 GPTs 对话。
6. 用户允许自动发送后，后续常规 GPTs 提示无需逐次确认；此权限不覆盖选题和发布关卡。

## 3. 策划冻结

要求同一个 GPTs 冻结并输出：

- 正式书名
- 原作版本与禁用设定
- 类型、读者合同与目标体量
- 总字数、章节数量、单章字数区间
- 主角、金手指、主要人物表与保护等级
- 对手机制与世界规则
- 故事主线、章节目录、每章任务、爽点和结尾钩子
- 叙事人称与文风

保存策划文本，使用 `relocate` 更新正式书名和书籍目录，并登记章节总数。此后不得由 Agent 擅自改动创意资产；需变更时交回同一 GPTs。

## 4. 逐章生产与并行 Word

对第 1 章到最终章循环：

1. 要求 GPTs 只输出当前章正文，并使用唯一的起止边界标记。
2. 等待完整结束标记后原样提取正文。
3. 保存当前章纯文本并立即更新章节断点。
4. 立即自动发送下一章要求。
5. 在 GPTs 生成下一章期间，制作上一章独立 DOCX。
6. 文件名使用 `第N章 章节名.docx`，保存到正式书名文件夹。
7. 机械检查文件存在、非空、可作为 ZIP/DOCX 打开、标题与章节数正确。
8. 不渲染图片，不逐页检查，不做字体、分页或视觉校验。
9. 记录纯文本路径、Word 路径和章节状态。

## 5. GPTs 全书复核

全部章节完成后，将最终全文交给同一个 GPTs 复核：

- 总字数与单章字数是否符合冻结范围
- 人物姓名、视角、规则、时间线和章节衔接是否一致
- 是否存在重复、遗漏、未回收伏笔或人物崩坏
- 结局是否完成情感与因果结算

所有文学修改必须由 GPTs 完成。若 GPTs 修改章节，替换对应纯文本并重新生成受影响的 DOCX，然后标记为 `finalized`。

## 6. 合并文件

1. 按最终章节顺序生成 `《书名》全书合并版.docx`。
2. 合并文件依次包含书名、各章标题和最终正文。
3. 机械检查章节标题顺序、总章数、文件存在且非空。
4. 将合并文件登记为唯一的番茄正文来源。

## 7. 番茄草稿

1. 使用 Chrome 打开配置中的番茄工作台地址；地址留空时从当前已登录会话进入，并核对非敏感账号提示。
2. 新建或恢复记录的短故事草稿。
3. 填写正式书名。
4. 从全书合并版提取完整内容，一次性写入正文编辑器。
5. 将 `第1章……` 等设置为番茄“章节标题”格式，核对识别出的章节总数。
6. 记录草稿 ID、URL、内容上传状态和章节格式状态。

## 8. 封面与元数据

1. 要求同一个 GPTs 根据最终小说生成竖版番茄小说封面。
2. 封面使用最终书名和配置中的 `author_name`，且不得包含平台水印或其他作者名。
3. 若书名、作者名或文字不可识别，要求同一 GPTs 重做，直至可用。
4. 保存封面到书籍文件夹并上传到番茄草稿。
5. 将“是否使用 AI”设置为“是”。
6. 根据番茄当时提供的分类，选择最匹配题材、原作和读者的主分类及辅助分类，并在完成报告中列出。
7. 设置试读比例为 `30%`。
8. 作者名称使用配置中的 `author_name`，并核对其与当前番茄账号的作者显示信息一致。

## 9. 完成与发布保护

1. 点击“存草稿”或确认自动保存完成。
2. 核对书名、正文、章节、封面、AI 标记、分类和试读比例。
3. 将断点推进到 `draft_complete_waiting_publish_authorization`。
4. 停止操作，不点击“下一步”中的提交审核动作，不发布、不签约。
5. 只有用户在新消息中明确授权审核或发布后，才可记录授权并执行对应动作。
6. 最终汇报所有文件路径、番茄草稿状态和明确的未发布状态。
