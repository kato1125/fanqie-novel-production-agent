# 番茄短篇 GPTs 一次直出工作流

## 目录

1. 独立配置与工程包
2. GPTs 对话与选题
3. 用户选题关卡
4. 策划冻结
5. 逐章一次生成与章节 Word
6. GPTs 最终合并版 Word
7. 番茄草稿
8. 封面与元数据
9. 完成与发布保护

## 1. 独立配置与工程包

1. 按 [configuration.md](configuration.md) 定位并验证用户自己的配置。
2. 接收用户在当前 Codex 对话上传的完整工程包 DOCX。
3. 将附件原样保存到配置的 `output_root` 下的新书临时目录。
4. 初始化断点，登记工程包、配置文件和书籍目录。
5. 不要求用户预先把工程包放入指定文件夹。

## 2. GPTs 对话与选题

1. 使用 Chrome 打开配置中的 GPTs 地址。
2. 核对非敏感账号提示，并在整本书中保持同一个 GPTs 对话。
3. 上传完整工程包。
4. 要求 GPTs 只生成完整选题列表，不写策划、大纲或正文。
5. 保存 GPTs 选题原文，不删减、不概括、不重排、不改写。

## 3. 用户选题关卡

1. 将全部选题原样展示给用户。
2. 将断点推进到 `awaiting_topic_confirmation`。
3. 必须暂停，等待用户明确选择。
4. 禁止根据评分、流行度或 Agent 判断代替用户选题。
5. 用户选择后，完整记录用户回复，并发送回同一个 GPTs 对话。
6. 用户的自动执行许可仅覆盖后续常规步骤，不覆盖选题和发布。

## 4. 策划冻结

让 GPTs 根据用户选题一次性完成并冻结：

- 正式书名
- 类型、读者合同和目标体量
- 章节总数及单章目标
- 主要人物、世界规则和核心机制
- 故事主线、章节目录、每章任务、爽点和结尾钩子
- 叙事人称与文风

保存策划原文，登记章节总数，并使用 `checkpoint.py relocate` 将临时目录迁移到
正式书名目录。策划之后不由 Agent 修改创意设定。

## 5. 逐章一次生成与章节 Word

对所有章节依次执行：

1. 向同一个 GPTs 请求当前章最终正文，并明确“这是唯一一次正文生成”。
2. 要求只输出本章标题和正文，带唯一的开始、结束标记。
3. 等待完整结束标记；未结束时继续等待同一次输出，不另发重新生成请求。
4. 将首次完整输出视为该章最终正文，不要求润色、扩写、缩写或重写。
5. 原样保存纯文本。
6. 本地机械创建 `第N章 章节名.docx`，不改变任何正文内容。
7. 只检查文件存在、非空、可作为 DOCX 打开、标题与章号匹配。
8. 更新章节断点后再请求下一章。
9. 若浏览器中已存在该章完整输出或本地已有章节 Word，恢复时不得重复生成。

本流程不执行全书复核，不设置复核步骤，也不因文学质量重新生成章节。

## 6. GPTs 最终合并版 Word

全部章节 Word 保存完成后：

1. 在同一个 GPTs 对话中要求生成一个可下载的最终合并版 DOCX。
2. 明确要求只按既有章节顺序机械汇总已经生成的章节。
3. 明确禁止润色、校对、复核、删改、补写、改名或重新生成正文。
4. 文件名使用 `《书名》GPTs最终合并版.docx`。
5. 下载并保存到正式书籍目录。
6. 本地只检查文件存在、非空、可打开、章节标题数量与顺序一致。
7. 不进行逐句比对或文学复核。
8. 若 GPTs 没有提供可下载 DOCX，继续要求其以 DOCX 文件交付；仍失败则暂停报告，
   不得调用本地合并程序替代。
9. 将该 DOCX 登记为唯一的 `fanqie_upload_source`。

## 7. 番茄草稿

1. 使用 Chrome 打开或恢复配置中的番茄工作台草稿。
2. 填写正式书名。
3. 优先使用平台的 Word 导入功能上传 GPTs 最终合并版。
4. 若页面要求粘贴正文，只能机械读取该合并版的内容后粘贴，不能使用聊天记录、
   章节草稿或本地重新合并文件。
5. 设置和核对章节标题格式与章节总数。
6. 保存草稿 URL、草稿 ID 和正文导入状态。

## 8. 封面与元数据

1. 若番茄草稿需要封面，要求同一个 GPTs 根据最终书名生成竖版封面。
2. 封面使用配置中的作者名，不含平台水印或其他作者名。
3. 保存并上传封面。
4. “是否使用 AI”设置为“是”。
5. 根据番茄当前选项选择最匹配的分类，并在汇报中列出。
6. 试读比例设置为 `30%`。

## 9. 完成与发布保护

1. 保存草稿并核对书名、章节、正文来源、封面及元数据。
2. 将状态推进到 `draft_complete_waiting_publish_authorization`。
3. 停止操作，不提交审核、不发布、不签约。
4. 只有用户在新消息中明确授权后，才执行相应公开动作。
